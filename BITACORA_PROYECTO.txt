1. OBJETIVO DEL PROYECTO
=========================================================================
Sistema web privado y auto-alojado "Multiagente de WhatsApp" para
coordinar soporte técnico interno en campo. 4 operadores humanos
gestionan conversaciones con técnicos en calle de forma simultánea.

3. TECNOLOGÍAS UTILIZADAS
=========================================================================
- Gateway: OpenWA (rmyndharis/OpenWA) en Podman
- Backend: Node.js + Express + Socket.io
- BD: SQLite con Prisma ORM
- Frontend: HTML5 + Tailwind CSS + Vanilla JS (ES Modules)
- Auth: JWT (jsonwebtoken + bcryptjs)
- Tiempo real: Socket.io (WebSockets)
- Contenedores: Podman + podman-compose

7. PROBLEMA ACTUAL (PENDIENTE DE RESOLVER)
=========================================================================
TÍTULO: Sesión de WhatsApp se queda en estado "authenticating"
         y no completa la conexión.

SÍNTOMAS:
- QR se genera correctamente en el Dashboard de OpenWA
- Usuario escanea el QR desde WhatsApp (teléfono)
- Estado pasa a "authenticating" pero nunca a "connected"
- No hay errores visibles en logs de OpenWA
- whatsapp-web.js 1.34.7 intenta auto-negociar versión de WhatsApp Web
  y falla silenciosamente

CAUSA PROBABLE:
Incompatibilidad entre whatsapp-web.js 1.34.7 y la versión actual
del servidor de WhatsApp. El auto-negotiation selecciona una versión
de WhatsApp Web que ya no es soportada por los servidores de Meta.

SOLUCIONES POSIBLES (en orden de probabilidad):

OPCIÓN A (Recomendada) - Fijar versión de WhatsApp Web:
  1. Editar OpenWA/docker-compose.dev.yml
  2. Agregar en environment del servicio openwa:
       - WWEBJS_WEB_VERSION=2.3000.1017314725
  3. Reconstruir: podman-compose -f docker-compose.dev.yml down
     y luego: podman-compose -f docker-compose.dev.yml up -d --build
  4. Crear nueva sesión y escanear QR
  5. Si falla, probar versiones alternativas:
     - 2.2412.54
     - 2.3000.1015901306
     - 2.3000.1015944858

OPCIÓN B - Actualizar whatsapp-web.js en el contenedor:
  - whatsapp-web.js 1.37.x tiene correcciones de autenticación
  - Editar package.json de OpenWA y fijar "@open-wa/wa-automate": "^1.37.0"
  - Reconstruir imagen

OPCIÓN C - Cambiar engine a Baileys:
  - Agregar en environment: ENGINE_TYPE=baileys
  - Baileys es más estable y no necesita Chromium
  - Reconstruir y probar

OPCIÓN D - Probar en otro teléfono/dispositivo:
  - Probar con otro teléfono Android o iOS actualizado
  - Verificar que WhatsApp esté en última versión desde la tienda

OPCIÓN E - Usar un proxy:
  - Configurar proxy para whatsapp-web.js si hay restricciones de red
  - Agregar en environment:
      - PROXY_HOST=...
      - PROXY_PORT=...

8. APIs DE OPENWA DOCUMENTADAS
=========================================================================
Health:             GET  /api/health/ready
Crear Sesión:       POST /api/sessions         {"name": "mi-sesion"}
Iniciar Sesión:     POST /api/sessions/{id}/start
Obtener QR:         GET  /api/sessions/{id}/qr
Estado Sesión:      GET  /api/sessions/{id}
Detener Sesión:     POST /api/sessions/{id}/stop
Eliminar Sesión:    DELETE /api/sessions/{id}
Enviar Texto:       POST /api/sessions/{id}/messages/send-text
                    {"chatId": "52123@c.us", "text": "Hola"}
Webhooks:           POST /api/sessions/{id}/webhooks
                    {"url": "...", "events": ["message.received"]}
Historial Chat:     GET  /api/sessions/{id}/messages/{chatId}/history
Auth:               Header: X-API-Key

9. FLUJO DE DATOS ESPERADO
=========================================================================
WhatsApp Phone
    ↓ (mensaje entrante)
OpenWA Gateway (puerto 2785)
    ↓ (webhook HTTP POST)
Backend /webhook/message (puerto 3000)
    ↓ (guarda en BD + emite WebSocket)
Frontend (todos los operadores conectados)
    ↓ (operador escribe respuesta)
Backend API /api/chats/:id/send
    ↓ (guarda en BD + POST a OpenWA)
OpenWA API /api/sessions/{id}/messages/send-text
    ↓ (envía a WhatsApp)
WhatsApp Phone del técnico

10. COMANDOS ÚTILES
=========================================================================
# Iniciar backend multiagente:
  cd ~/Escritorio/whatsappumsr/backend && npm start

# Iniciar OpenWA:
  cd ~/Escritorio/whatsappumsr/OpenWA
  podman-compose -f docker-compose.dev.yml up -d --build

# Ver logs de OpenWA:
  podman logs -f openwa-api

# Simular mensaje entrante (prueba sin WhatsApp):
  curl -X POST http://localhost:5000/webhook/message \
    -H "Content-Type: application/json" \
    -d '{"event":"onMessage","data":{"chatId":"521234567890@c.us","sender":{"name":"Luis Técnico"},"body":"Necesito soporte"}}'

# Verificar conexión OpenWA desde backend:
  curl -s http://localhost:5000/api/openwa/status \
    -H "Authorization: Bearer $(TOKEN)"

# Login de prueba:
  curl -s -X POST http://localhost:5000/api/auth/login \
    -H "Content-Type: application/json" \
    -d '{"username":"carlos","password":"operador123"}'

# Enlaces:
  Frontend:  http://localhost:5000
  OpenWA:    http://localhost:8002 (API) / http://localhost:2886 (Dashboard)
  Swagger:   http://localhost:8002/api/docs
